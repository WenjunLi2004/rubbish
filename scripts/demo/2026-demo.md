# 2026 试题演示脚本（答题纸截屏用）

> 在 euler 容器上、以 `liwenjun2023150001` 登录、`PS1='[\u@\h \W]\$ '`。
> 自动冒烟见 `make smoke`（`scripts/smoke/smoke-2026.sh`）。本文给的是**多终端手工演示**
> 的命令序列，便于逐张截屏。日志位于 `~/log/chat-logs`（`LOGFILES`）。

约定：`CONF=~/ChatRoom/config/chatserver.conf`，二进制 `~/ChatRoom/bin/chatserver_lwj_1.0`、
`~/ChatRoom/bin/chatclient`。

## 0. 构建

```bash
cd ~/ChatRoom && make clean && make
```

## 1. 启动守护进程（终端 S，用 sudo 让日志归 root）

```bash
sudo ~/ChatRoom/bin/chatserver_lwj_1.0 ~/ChatRoom/config/chatserver.conf
# 守护化后立即返回提示符
ps -ef | grep chatserver_lwj_1.0 | grep -v grep      # 截屏：PPID=1、TTY=?
ls -l ~/Server/fifo                                   # 截屏：4 个 p 开头的 FIFO
ls -l ~/log/chat-logs/server/                         # server.log / threads.log，0600 root
sudo head -n 20 ~/log/chat-logs/server/server.log     # 启动时间 + thread pool created: 100 threads
```

## 2. 注册（终端 A/B，演示重名拒绝）

```bash
# 终端 A
~/ChatRoom/bin/chatclient $CONF liwenjun_1 pw1
chat> /register            # OK: register ok

# 终端 B
~/ChatRoom/bin/chatclient $CONF liwenjun_2 pw2
chat> /register            # OK: register ok
chat> /register            # 再来一次 -> ERROR: username already exists（重名拒绝）
```

## 3. 登录（在线人数 + 名单；失败登录）

```bash
# 终端 A
chat> /login               # OK: login ok; online N: ... 并收到 [online N] 名单
# 终端 B
chat> /login               # 终端 A 会收到 [system] liwenjun_2 is now online 与刷新名单

# 终端 C：未注册用户登录失败
~/ChatRoom/bin/chatclient $CONF liwenjun_3 pw3
chat> /login               # ERROR: username does not exist
```

## 4. 一对一发送 5 次（成功日志）

```bash
# 终端 A，连发 5 次
chat> /send liwenjun_2 How are you, liwenjun_2     # 重复 5 次
# 终端 B 实时收到 5 条 [liwenjun_1] How are you, liwenjun_2
sudo grep "(liwenjun_1, liwenjun_2" ~/log/chat-logs/server/server.log   # 5 条 ..., sent
```
> 重要朋友标记：当成功次数 **超过 5**（第 6 次起），发送方一侧的应答与在线名单里
> 目标显示为 `liwenjun_2*`。

## 5. 登出广播 + 离线消息暂存与回推

```bash
# 终端 B
chat> /logout              # 终端 A 收到 [system] liwenjun_2 has logged out + 刷新名单
# 终端 A：向已离线的 liwenjun_2 发消息 -> 暂存
chat> /send liwenjun_2 Hi, let's play badminton?   # ERROR: ... stored as pending
sudo grep "pending" ~/log/chat-logs/server/server.log
# 终端 B：重新登录 -> 收到带原始时间的离线消息
chat> /login               # [offline from liwenjun_1 @ <原始时间>] Hi, let's play badminton?
```

## 6. 机器人管理

```bash
# 终端 A
chat> /bot add 2           # OK: added 2 bot(s): lwjbot_..,lwjbot_.. 并在名单中出现机器人
chat> /send lwjbot_<x>_<y> 你好     # 收到 [lwjbot_..] 幸会，liwenjun_1，很高兴认识您
chat> /bot del 1           # OK: removed 1 bot(s): ...；在线用户看到 [system] robot ... has logged out
sudo grep "bot" ~/log/chat-logs/server/server.log   # 机器人 注册/登录/消息/退出 记录
```
> 机器人用户名随机、可能永不再登录，故服务器**不为离线机器人保留离线消息**；
> 相关丢弃在日志中明确记录。

## 7. 线程池活动 + 收尾

```bash
sudo tail -n 40 ~/log/chat-logs/server/threads.log   # dispatch(busy)/recycle(idle) 时间戳
sudo cat  ~/log/chat-logs/server/server.log          # 注册/登录/发送/登出/离线回推/线程管理 全量
# 关闭：向 daemon 发 SIGTERM
sudo kill -TERM $(pgrep -f 'chatserver_lwj_1.0')
ls ~/Server/fifo                                      # 4 个公共 FIFO 已清理
ls -l /dev/shm/chatroom_lwj_users 2>&1                # 共享对象已被 shm_unlink
sudo grep "thread pool shut down" ~/log/chat-logs/server/server.log
```

## 关于 `/var/log/chat-logs` 的不一致

试题正文定义 `LOGFILES=~/log/chat-logs/`，但有一条示例命令写 `cat /var/log/chat-logs/...`。
本实现以显式的 `LOGFILES=~/log/chat-logs/` 为准。若答题需要 `/var/log/chat-logs` 路径，
可建一个 **root 拥有的软链接**（可选，代码不依赖它）：

```bash
sudo mkdir -p /var/log
sudo ln -s ~/log/chat-logs /var/log/chat-logs    # 仅为兼容示例命令；非必需
```

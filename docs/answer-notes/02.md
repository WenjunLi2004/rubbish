# 答题纸素材 02（对应 1.1) ps 看 daemon 6 分 + 1.2/b 日志 5 分）

服务器要长期驻留、对所有客户端持续服务，因此必须**守护进程化**：脱离启动它的终端，
即使关闭 ssh 会话或终端，进程也不被 `SIGHUP` 杀死，仍在后台监听 4 个公共 FIFO。

守护化采用经典五步法：①`fork()` 后父进程立即退出，子进程不再是进程组组长，
`setsid()` 才能成功；②`setsid()` 新建会话与进程组，彻底脱离控制终端；
③设置信号处置（`SIGINT`/`SIGTERM` 记录日志并清理 FIFO 后退出，`SIGHUP`、
`SIGPIPE` 忽略）；④`chdir("/")`，避免占用启动目录所在文件系统使其无法卸载；
⑤把 `stdin`/`stdout`/`stderr` 重定向到 `/dev/null`。`umask(0)` 紧接 `fork`
之后设置，保证随后 `mkfifo(0666)`、`mkdir` 的权限位不被默认 umask 砍掉。

正因为第⑤步切断了标准流，守护化之后任何 `printf`/`perror` 都被写进 `/dev/null`
而**不可靠**——看不到、也无处可查。因此本阶段引入统一日志函数 `log_message()`，
把启动、生效配置、FIFO 路径、协议结构体尺寸、注册/登录/登出/聊天事件、各类致命
错误统一写入 `/var/log/chat-logs/server/server.log`。该文件以
`O_CREAT|O_APPEND|O_WRONLY`、权限 `0600` 打开；服务器以 `sudo`（root）启动，
日志保持 root 所有、不 chown 回普通用户，体现"日志只许特权进程写"。
日志行格式为 `YYYY-MM-DD HH:MM:SS [pid=12345] [LEVEL] message`，含时间戳与 pid。

验证方法：`ps -ef | grep chatserver_lwj_1.0.0` 可见进程 PPID 为 1（已被 init 收养）、
无控制终端（TTY 列为 `?`），证明守护化成功；`sudo cat /var/log/chat-logs/server/server.log`
能看到 `server starting`、`... ready`、`registered/login/logout user` 等条目，
证明日志链路与各业务事件均按设计落盘。

# 答题纸素材 01（对应 1.2/a 4 分）

服务器二进制命名为 `chatserver_lwj_1.0.0`，借鉴 Linux 内核 `x.y.z`
版本号约定：`x` 是主版本（协议大变更才递增），`y` 是次版本（约定偶数
为稳定版、奇数为开发版），`z` 是补丁号。本项目基础版选 `1.0.0`，
阶段 9 引入线程池后升至 `1.2.0`，明确表达"协议兼容、新增稳定特性"。

4 个公共 FIFO 的名字 `lwjregister`、`lwjlogin`、`lwjsendmsg`、`lwjlogout`
全部由 `config/chatserver.conf` 中的 `fifo_prefix = lwj` 派生而来，
未在 C 源里硬编码。这样换用户或换前缀只需改配置，体现了
"路径/命名 与 代码 解耦"的设计原则——也是试题 1.1.2 的要求。

`ls -l` 输出每行最左侧的"文件类型 + 权限"段中，第一个字符表征文件
类型：`-` 普通文件、`d` 目录、`l` 符号链接、`c` 字符设备、`b` 块设备、
`s` 套接字，而 `p` 正是 **named pipe（FIFO）** 的标志。因此对 4 个
FIFO 跑 `ls -l data/server_fifo/` 应看到 4 行 `p` 起首，是证明
"管道已建立"的最直观证据。

服务器 `mkfifo` 之前必须先 `umask(0)`：进程默认 umask 通常是 `0022`，
内核创建文件时会执行 `mode & ~umask`，把请求的 `0666` 砍成 `0644`，
导致非 owner 用户写入失败（客户端 `open(... O_WRONLY)` 返回 EACCES）。
显式 `umask(0)` 保留全部权限位，再交给 `mkfifo(path, 0666)`，
才能真正得到 `prw-rw-rw-` 的多写者公共管道。
